# schema/

DB 스키마 데이터 레이어. 스키마 빌드·검색·검증 등 **DB 스키마 자체**를 다루는 모듈만 포함한다.

## 모듈

| 파일 | 역할 | 호출 시점 |
|------|------|-----------|
| `build.py` | 스키마 JSON → TEI 임베딩 → `.npz` 저장 | 일회성 (스키마 변경 시) |
| `search.py` | 자연어 질의 → 스키마 임베딩 검색 (top-K) | 런타임 (매 질의) |
| `validator.py` | SQL에 사용된 테이블·컬럼이 실제 DB에 존재하는지 검증 | 런타임 |
| `value_hints.py` | 스키마 컬럼 타입 기반으로 DB 실제 값(DISTINCT, min/max) 조회 | 런타임 |
| `extract.py` | 스키마 정보 추출용 Pydantic 모델 | 런타임 |

## 원칙

- **DB 스키마와 직접 관련된 로직만** 이 폴더에 둔다.
- SQL 문자열 처리, LLM 호출 등은 `services/`에 배치한다.
